# Auth Service

Отвечает за регистрацию, авторизацию, управление сессиями и выпуск JWT-токенов.

## JWT

Используется модель:

- **Stateless Access Token**
- **Stateful Refresh Token (с ротацией)**

### Access Token
- Формат: JWT
- Алгоритм: **RS256**
- Короткий TTL (`AUTH_ACCESS_TOKEN_TTL`)
- Проверяется другими сервисами локально по **публичному ключу**
- Не хранится в БД

### Refresh Token
- JWT (RS256)
- Долгий TTL (`AUTH_REFRESH_TOKEN_TTL`)
- Привязан к серверной сессии
- В БД хранится только `hash(refreshToken)`
- При refresh происходит **ротация токена**


## Ротация токенов

При каждом login/register создаётся новая сессия.

Хранятся поля:
- sessionID
- userID
- expiresAt
- revoked
- tokenHash

Использованный **refresh token** инвалидируется, к физической сесси привязывается новый токен.
Производится это благодаря дедупликации сессий по **device_fingerprint**.
Это позволяет пользователю видеть в списке сессий только реальные физические сессии своего аккаунта.

## Session managment

Реализован полный функционал для управления сессиями.

Пользователь может выполнить:
- **Revoke Session**: 
    Завершить одну из активных сессий на другом устройстве
- **Logout All**:
    Завершить все свои активные сессии

## Очистка revoked сессий
При получении соответствующего события производится **очистка старых завершенных сессий**. 

За публикацию события отчистки старых сессий отвечает `scheduler-service`. Конфигурация частоты очистки и времени удержания старых сессий производится там.


## Интеграция

Сервис подписан на Kafka топик `auth.events` и слушает событие `session_cleanup`.

Другие сервисы (Gateway, Booking и др.):

- получают публичный RSA-ключ
- валидируют access token локально
- не обращаются к auth-service для проверки токена

## Безопасность

- Асимметричная подпись (RS256)
- Stateless access
- Refresh rotation
- Session revoke
- Хранение только hash refresh токена

## API

- POST `/auth/register` -  Регистрация пользователя
- POST `/auth/login` - Авторизация пользователя
- POST `/auth/refresh` - Обновление access/refresh токенов
- POST `/auth/logout` - Выход пользователя (инвалидация refresh - токена)
- GET `/users/me` - Получить профиль текущего пользователя
- GET `/users/sessions/active` - Получить активные сессии - пользователя
- GET `/users/sessions/all` - Получить все сессии пользователя
- POST `/users/sessions/revoke` - Отозвать (разлогинить) сессию по ID

Подробнее в [swagger](../docs/swagger.yaml).
