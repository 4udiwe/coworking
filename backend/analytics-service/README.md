# Analytics Service
Сервис аналитики, отвечающий за сбор, обработку и агрегацию данных о бронированиях в системе коворкингов.

Основная задача — предоставление быстрой и дешёвой аналитики в реальном времени без нагрузки на transactional сервисы.

## ClickHouse

Сервис использует ClickHouse в качестве OLAP-хранилища и совершает предагрегацию данных через **materialized views**.

### Data Pipline

1. **Kafka**

    Сервис слушает топик бронирований, накапливает собыитя в **in-memory buffer** для одномоментной вставки батчем (более эффективно c ClickHouse)

2. Таблица `booking_events` - event log, который отражает все изменения.
3. Таблица `booking_state` - хранит последнюю версию состояния бронирования через `ReplacingMergeTree`

### Предагрегация

Вся аналитика строится через materialized views, а не runtime-запросы.

- `coworking_hourly` - Почасовая загрузка:
    - показывает, сколько бронирований приходится на каждый час
    - учитывает длительность бронирования через arrayJoin(range(...))

- `coworking_weekday` - Популярность по дням недели:
    - показывает число бронирований распределенное по дням недели
    - агрегация по toDayOfWeek

- `coworking_heatmap` - Heatmap коворкинга:
    - двумерная модель: weekday × hour
    - основа для визуализации загрузки коворкинга. Удобно для анализа администратором

- `place_heatmap` - Heatmap рабочего места:
    - двумерная модель: weekday × hour
    - позволяет анализировать конкретные места

## Интеграция

Сервис подписан на Kafka топик `booking.events`. Более подробно в [event-catalog](../docs/event_catalog.md)

Использует публичный RSA-ключ для валидации access token входящих HTTP запросов.
Путь к файлу с ключем (pubilc.pem) указывается в [config.yaml](config/config.yaml)

## Конфигурация

Через `.env`. Для запуска можно скопировать `.env.example`
